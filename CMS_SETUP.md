# CMS setup

Проектът вече има общ админ панел за двата сайта:

- `/admin` - Decap CMS панел
- `content/apartment.json` - редактирани данни за апартамента
- `content/bistro.json` - редактирани данни за Бистро Колибаре

## Какво може да редактира клиентката

### Апартамент

- WhatsApp номер
- цени в евро
- линк към обявата
- линк към сайта на бистрото
- снимки на апартамента
- снимки на Бистро Колибаре, които се показват в страницата на апартамента

### Бистро Колибаре

- телефон, WhatsApp, email, Facebook
- бележка към менюто
- категории и ястия в менюто
- снимки в галерията

## Как се активира login

В Netlify трябва да се включат:

1. Site settings -> Identity -> Enable Identity
2. Identity -> Registration preferences -> Invite only
3. Identity -> Services -> Git Gateway -> Enable Git Gateway
4. Invite users -> поканете клиентката с нейния email

След това клиентката влиза на:

`https://your-domain.com/admin`

## Важно за структурата

Най-чистият вариант е двата публични сайта да се публикуват от един общ Netlify проект:

- `/Апартамент/`
- `/Ресторант/`
- `/admin/`

Така един login управлява и двата сайта.

## Локална разработка

Публичните страници четат JSON файловете чрез `fetch`, затова за тест трябва да се пуска сървър от корена на проекта, не само от папката `Апартамент`.

Пример:

```powershell
python -m http.server 8088 --bind 127.0.0.1
```

После:

- `http://127.0.0.1:8088/Апартамент/`
- `http://127.0.0.1:8088/Ресторант/`
- `http://127.0.0.1:8088/admin/`
